# 快速开始

ObjectDocs 是一个基于 Next.js 14 和 Fumadocs 的现代文档引擎，支持“配置即代码”的理念。本指南将帮助你快速搭建一个新的文档项目。

## 准备工作

确保你的环境满足以下要求：

- **Node.js**: 18.17 或更高版本
- **包管理器**: 推荐使用 pnpm (npm 或 yarn 也可以)

## 1. 初始化 ObjectDocs

ObjectDocs 可以在任何现有项目中初始化，或作为新的独立项目。

<Steps>
### 方式 A: 使用 npm init (推荐)

最快速的入门方式：

```bash title="Terminal"
mkdir my-docs
cd my-docs
npm init @objectdocs
```

这将自动运行初始化命令并设置所有必需的内容。

### 方式 B: 使用 npx

在现有项目中初始化：

```bash title="Terminal"
cd your-existing-project
npx @objectdocs/cli init
```

这将：
- 从 `@objectdocs/site` 复制站点引擎到 `content/.objectdocs`
- 创建 `content/package.json` 并添加脚本 (`dev`, `build`, `start`)
- 在 `content/.objectdocs` 中安装依赖
- 将 `content/.objectdocs` 和 `content/node_modules` 添加到 `.gitignore`
- 保持项目根目录整洁

### 创建内容目录

如果尚未创建：

```bash title="Terminal"
mkdir -p content/docs
```
</Steps>

## 2. 添加内容

ObjectDocs 使用文件系统路由。所有的文档都存放在 `content/docs` 目录下。

<Steps>
### 创建首页

新建 `content/docs/index.mdx`：

```mdx title="content/docs/index.mdx"
---
title: 欢迎
description: 你的新文档站点
---

# Hello World

欢迎使用 ObjectDocs！这是你的首页。
```

### 配置侧边栏

新建 `content/docs/meta.json` 来定义页面顺序：

```json title="content/docs/meta.json"
{
  "pages": ["index"]
}
```

### 配置站点信息

新建 `content/docs.site.json` 来配置站点名称和导航：

```json title="content/docs.site.json"
{
  "branding": {
    "name": "My Docs"
  },
  "links": [
    {
      "text": "Home",
      "url": "/"
    }
  ]
}
```
</Steps>

## 3. 启动开发服务器

进入 content 目录并启动开发服务器：

```bash title="Terminal"
cd content
npm run dev
```

开发服务器将：
- 在端口 7777 上启动（默认）
- 监听 `content/docs.site.json` 的变化并自动重启
- 热重载你的 MDX 内容更改
- 复制 `public/` 资源到站点引擎

现在访问 [http://localhost:7777](http://localhost:7777)，你应该能看到你的文档站点了！
